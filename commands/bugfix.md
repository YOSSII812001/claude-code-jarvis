# ユーザーが提供したバグの説明から問題を修正する: $ARGUMENTS

## ゴール

問題を再現し、バグを修正する

## ステップ

ユーザーが提供したバグの説明: $ARGUMENTS

1. バグの説明を理解し、根本原因を考える。問題の再現を試みる。
2. エラーを修正する。MCPツール `context7` と `brave-search` を使って最新のライブラリ情報を取得しエラーを理解する。これらのMCPツールの検索は複数のサブエージェントで行う。また問題の調査にはネイティブのウェブ検索サブエージェントを使用する。
3. 別のエラーが発生した場合は、常にこの 1 と 2 のデバッグ作業を繰り返す。
4. 問題が解決した際に、バグの原因と使用した解決方法を出力する
