# ウサコンチェックリスト コードチェック & E2Eテスト: $ARGUMENTS

## ゴール

Google スプレッドシートのウサコンチェックシートを、**マネージャー（自分）が統括**し、コードチェックは**サブエージェントに委任**、E2Eテストは**ブラウザ操作で直接実行**する。判定に詰まった場合は**Codex（OpenAI）にセカンドオピニオン**を求め、ユーザーの手を止めずに一気通貫でチェックを完了する。チェック通過時にA列を✅、不備時はGitHub Issueを作成する。

## 引数

- `$ARGUMENTS` にセクション番号を指定 → そのセクションのみチェック
  - 例: `/checklist 5` → ダッシュボードセクション（4項目）のみ
- 引数なし → 未チェック項目を先頭から順にチェック

## 手順

### ⓪ スキルのコンテキスト読み込み

フロー開始時に以下のスキルを**Readツールで読み込み**、操作ナレッジをコンテキストに取り込む：

1. **usacon-checklist** (`~/.claude/skills/usacon/checklist.md`) — チェックリスト親スキル（共通定義・コマンド分岐・テストアカウント・GitHub Issue作成）
   - コードチェック詳細: `~/.claude/skills/usacon/references/checklist-code.md`
   - E2Eテスト詳細: `~/.claude/skills/usacon/references/checklist-e2e.md`
   - アプリUI操作ガイド: `~/.claude/skills/usacon/references/app-knowledge.md`
2. **google-sheets-mcp** (`~/.claude/skills/google-sheets-mcp/SKILL.md`) — スプレッドシート操作パターン（セルナビゲーション・値読み取り・編集・シートタブ切替・ドロップダウン操作）

### ① 環境準備

1. `tabs_context_mcp` でタブ確認
2. スプレッドシートタブを特定（なければナビゲート）
3. 「アプリ」シートが選択されていることを確認
4. usaconプレビューサイト用の新タブを作成
5. https://preview.usacon-ai.com にナビゲート → ログイン（usacon/checklist.mdに認証情報あり）

### ② チェックループ（マネージャー主導）

セクション番号が指定されている場合はそのセクションの行範囲のみ、指定なしの場合は未チェック行を先頭から順に：

1. **行データ読み取り**: スプレッドシートタブ → 名前ボックスで対象行にジャンプ → B列・C列・F列を読み取り
2. **スキップ判定**: 水色背景セクションヘッダー行 / A列に既にチェック済みの行
3. **コードチェック委任**: Taskツールでサブエージェント起動（usacon/references/checklist-code.md のワーカープロンプトテンプレート参照）
   - subagent_type: "general-purpose"
   - B列・C列・F列の内容をプロンプトに埋め込み
4. **Codexエスカレーション**（必要時のみ）: ワーカーが UNSURE を返した場合、または判定に迷う場合
   - codexスキルの手順でセカンドオピニオンを取得（Taskツール Bashサブエージェント経由）
   - ワーカー分析 + Codex判定を総合して最終判定
5. **結果処理**:
   - OK → A列✅に書き換え + G列に担当者「竹下」を選択
   - NG → D列に「NG」、E列に原因記入 → GitHub Issue作成 → F列にIssue番号記入 → G列に担当者選択
6. **E2Eテスト**（必要な項目のみ）: usaconタブで手順に基づきブラウザテスト → D列に結果記入
7. 次の行へ

### ③ 完了報告

チェック完了後、結果サマリーを報告：
- 総チェック数 / 成功数 / 失敗数 / スキップ数
- 失敗した項目のリスト（行番号・項目名・原因）
- 作成したGitHub Issueのリスト（Issue番号・タイトル・URL）
