description: E2Eテスト合否は「通過数」ではなく「変更箇所×テスト対応マッピング」で判定。feat種別ではCodexにカバレッジ十分性評価を必ず依頼。
type: feedback

E2Eテスト完了時は、変更ファイルごとのテストマッピング表を作成してから合否判定する。

**Why:** Issue #1534で7/7 PASSでも3箇所が未検証だった。fileStreamHandler.js（BLOCKER修正）のファイル添付フロー、非ストリーミング経路、Web検索がテストされていなかった。Codex（QAエンジニア役）が「NO」判定を出して初めて発覚。

**How to apply:**
- feat種別のIssueでは、Codexに「E2Eカバレッジ十分性評価」を必ず依頼する
- 変更ファイル × テスト項目のマッピング表を作成し、全変更箇所にテストが対応していることを確認
- モニタリング/ログ系の変更はユーザー影響なしとして分離評価（本番で確認可能）
