type: feedback

プラグインとカスタムスキルは「置き換え」ではなく「レイヤー分離」で共存させる。

**Why:** Codexプラグインテスト(2026-03-31)で判明。9個の依存スキルがカスタムの `codex exec` パターンに直接依存しており、プラグインへの一括移行は依存チェーンを破壊する。一方、プラグインには構造化レビューやジョブ管理など独自の強みがある。

**How to apply:**
- 新しいプラグイン/ツール導入時、まず「既存スキルの依存関係マップ」を作成する（今回は9スキルが該当）
- 依存スキルが多い機能は維持し、新機能を「上位レイヤー」として追加する
- 役割分担表をスキル冒頭に明記し、どの用途でどちらを使うか迷わないようにする
- プラグイン固有のベストプラクティス（gpt-5-4-promptingのXMLブロック等）はカスタムスキルに取り込む
- Windows環境ではプラグイン更新後に `spawn` の `shell` オプションを再確認する
