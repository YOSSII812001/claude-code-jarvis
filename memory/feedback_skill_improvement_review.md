スキルファイル（SKILL.md）の編集後、Codex read-onlyレビューを必ず実行すること。

**Why:** スキル改善のたびに以下のような「しょうもない見落とし」が発生している:
- テーブル内のパス表記が既存行の規約と不一致（相対パス vs 絶対パス、スキル内参照 vs スキル間参照）
- マークダウンテーブルの列数ズレ
- 改訂履歴の日付・内容の不整合
- 複数ファイルに分散挿入する際の挿入位置ミス

**How to apply:**
1. スキルファイルの編集完了後、**変更をCommitする前に**Codex read-onlyでレビューを実行する
2. レビュー観点: 誤字脱字、テーブル構文、パス表記の一貫性、既存セクションとの矛盾、YAML frontmatter
3. 特に**スキル間クロスリファレンス**（別スキルのファイルを参照する行）は、そのテーブル内の他の行のパス規約と一致しているか重点チェック
4. 例: `references/xxx.md`（スキル内相対パス）が並ぶテーブルに `codex/SKILL.md` と書くと、同スキル内の `codex/` ディレクトリと誤解される → `~/.claude/skills/codex/SKILL.md` と書く
