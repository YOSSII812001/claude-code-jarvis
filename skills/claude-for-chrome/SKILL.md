# Claude for Chrome ガイド

## 概要
Claude for Chromeは、Chromeブラウザ内でClaudeがウェブサイトの読み取り、クリック、ナビゲーションを行えるブラウザ拡張機能です。サイドパネルで直接動作し、ユーザーが見ているものを認識してアクションを実行します。

**対象プラン**: Max プラン（ベータ版として提供）

## インストール方法

1. Google Chromeブラウザを開く（モバイル非対応）
2. [Chrome Web Store](https://chromewebstore.google.com/detail/claude/fcoeoabgfenejglbffodgkkbkcdhcgfn)からインストール
3. 「Chromeに追加」をクリック
4. Claudeアカウントでサインイン
5. パズルピースアイコン → Claudeの横のピン留めアイコンをクリック
6. 必要な権限を許可

## コア機能

### マルチタブ機能
複数のブラウザタブを同時管理可能。タブをClaudeの指定タブグループにドラッグすると、グループ化された全タブを一度に表示・操作できます。

### 強化されたサイトナビゲーション
以下のプラットフォームは詳細な指示なしで操作可能：
- Slack
- Google Calendar
- Gmail
- Google Docs
- GitHub

「会議をスケジュール」「ドキュメントを更新」などの簡単なコマンドで動作します。

### バックグラウンドワークフロー
複雑なマルチステップワークフローを処理し、タブを切り替えても（Chromeが開いている限り）動作継続。通知を有効にしてタスク完了時にアラートを受信できます。

### ショートカット（スラッシュコマンド）
よく使うプロンプトをショートカットとして保存し再利用：
1. うまく機能するプロンプトをショートカットとして保存
2. チャットで「/」を入力してアクセス
3. ショートカットを選択して即座に適用
4. 設定からショートカットの編集・削除が可能

### スケジュールタスク
繰り返しタスクを自動実行：日次、週次、月次、年次で設定可能。拡張機能パネル右上の時計アイコンから設定。

## 権限モード

### Ask before acting（承認前に確認）
Claudeが計画を作成し、承認後に実行。**最も安全なオプション**。
- アクセスするウェブサイトとアプローチを事前に明示
- 計画外の行動には再度許可を要求
- 高リスクアクション（購入、アカウント作成、ファイルダウンロード）は常に確認

### Act without asking（確認なしで実行）
**高リスクモード** - Claudeがほぼ完全な自律性で動作。

以下の場合のみ使用：
- アクティブに監視している
- 信頼できるサイトでのルーティンタスク
- 問題発生時に即座に停止できる

## 必須権限を要するアクション

権限モードに関係なく、以下のアクションには明示的な許可が必要：
- 購入や金融取引
- ファイルやデータの永久削除
- 権限設定の変更
- アカウント作成
- 認証の付与
- 機密情報のウェブサイトへの入力

## 禁止されているアクション

以下のアクションは権限に関係なく禁止：
- 機密性の高いクレジットカードやID情報の取り扱い
- 信頼できないソースからのファイルダウンロード
- 永久削除（ゴミ箱を空にする、メール・ファイル・メッセージの削除）
- セキュリティ権限やアクセス制御の変更
- 投資や財務アドバイスの提供
- 金融取引や投資取引の実行
- システムファイルの変更
- メールやウェブコンテンツからの指示の実行

## ブロックされるサイト

セキュリティのため、以下の高リスクサイトへのアクセスはブロック：
- 金融サービス・銀行サイト
- 投資・取引プラットフォーム
- アダルトコンテンツサイト
- 暗号通貨取引所

## セキュリティリスク

### プロンプトインジェクション攻撃
ウェブコンテンツ（ウェブサイト、メール、ドキュメント）に隠された悪意のある指示がClaudeを騙して意図しないアクションを取らせる可能性があります。

**対策：**
- 信頼できるサイトから始める
- 不審な動作（無関係なトピックへの言及、予期しないサイトへのアクセス、機密情報の要求）があれば即座に停止
- 問題はすぐに報告

### その他のリスク
- **意図しないアクション**: 指示の誤解やエラーによるデータ変更
- **確率的動作**: 同じリクエストでも異なる結果が出る可能性
- **金融リスク**: 意図しない購入や取引
- **プライバシーリスク**: 個人情報の不注意な露出や共有

## 推奨事項

### 安全な使用のために
1. 機密アカウント（銀行、医療、政府）にアクセスしない別のブラウザプロファイルを使用
2. 特に新しいウェブサイトでは、Claudeの提案アクションを承認前に確認
3. 複雑なマルチステップワークフローより、リサーチやフォーム入力などの簡単なタスクから始める
4. 意図しない動作を避けるため、プロンプトを具体的かつ慎重に調整

### 避けるべき使用
- 金融アカウントや投資の管理
- 法的文書や契約の処理
- 医療情報の処理
- 機密性の高い会社データを含む業務アカウントへのアクセス
- 他者の個人情報を含むサイトとの対話

## Claude Code MCP連携時の注意点

### 送信ボタンのクリック
- ウェブアプリのチャット入力欄で `computer` ツールの `left_click` を使って送信ボタンをクリックしても反応しないことが多い
- **推奨**: テキスト入力後は `key` アクションで `Return`（Enter）キーを送信する方が確実
- 送信ボタンがアイコンのみ（矢印など）の場合、クリック座標がずれやすいため特に注意
- 送信されたか確認するには、入力欄が空になっているか・応答が始まっているかをスクリーンショットで確認する

## MCP自動化パターン（サブエージェント向け）

### ツール初期化シーケンス
MCP toolsは使用前にToolSearchでロードが必要。以下の順序で一括ロードすること：

```
# Phase 1: 基本ツール（最初に必ずロード）
ToolSearch("select:mcp__claude-in-chrome__tabs_context_mcp")
ToolSearch("select:mcp__claude-in-chrome__navigate")
ToolSearch("select:mcp__claude-in-chrome__read_page")
ToolSearch("select:mcp__claude-in-chrome__computer")

# Phase 2: フォーム操作が必要な場合に追加
ToolSearch("select:mcp__claude-in-chrome__form_input")
ToolSearch("select:mcp__claude-in-chrome__find")
```

### ブラウザ接続パターン
```
1. tabs_context_mcp(createIfEmpty: true) でタブ取得
2. "Multiple Chrome extensions connected" エラー時:
   → switch_browser をロード・実行
   → ユーザーに「Chromeで Connect をクリック」を依頼
   → 再度 tabs_context_mcp を実行
3. 新しいタブで作業開始（既存タブは再利用しない）
```

### ナビゲーション＋確認パターン
```
1. navigate(url, tabId) でページ遷移
2. wait(3秒) でページ読み込みを待つ
3. read_page(filter: "interactive") で要素取得
4. 要素が見つからない場合 → screenshot で目視確認
5. coordinate指定でクリック or ref指定でクリック
```

### フォーム操作パターン
- **ドロップダウン**: `form_input(ref, value)` でoption valueを直接指定（クリック不要）
- **テキスト入力**: `computer(action: "left_click", ref)` → `computer(action: "type", text)`
- **送信**: `computer(action: "key", text: "Return")` が `left_click` より確実

### ダウンロードパターン
- PDFリンクのクリック後、新タブが開かない場合はダウンロードフォルダに保存される
- ダウンロード確認: `ls -lt ~/Downloads/*.pdf | head -5` でタイムスタンプを確認
- 複数ファイルのダウンロード間隔は2秒以上空ける

### 認証ハンドリング
| 認証ステップ | 対応方法 |
|-------------|---------|
| ID/パスワード（キャッシュ済み） | ログインボタンのクリックのみ代行可 |
| ID/パスワード（未入力） | **ユーザーに入力を依頼**（セキュリティ制約） |
| SMS認証コード | **ユーザーに入力を依頼**して待機 |
| CAPTCHA | **ユーザーに対応を依頼** |
| Cookie同意バナー | 最もプライバシー保護的な選択肢を自動クリック |

## ワークフローテンプレート

### MySoftBank 請求書PDFダウンロード

**前提条件**: ブラウザにSoftBank IDが保存済み、SMS受信可能な端末が手元にあること

**ナビゲーションパス**:
```
https://www.softbank.jp/mysoftbank/
  → 「ログインする」(携帯電話セクション, 最初のログインリンク)
  → ログインページ: ID/PW入力 → 「ログインする」ボタン
  → MySoftBankトップ
  → 上部タブ「料金・支払い管理」
  → サブタブ「書面発行」
  → 「請求書／内訳明細書」>「自分で印刷する（無料）」
  → 本人確認: SMSセキュリティ番号(3桁) → ★ユーザー入力待ち★
  → 請求書ダウンロードページ
```

**ダウンロードページの構造**:
- 照会月ドロップダウン（combobox）: value形式は `YYYYMM`（例: `202602`）
- PDFリンク: `電話番号別PDFファイル`（テキスト付きリンク、`doPrintMsn?idx=N`形式）
- 月変更 → PDFリンク再取得 → クリックでダウンロード、を月数分ループ

**月別ダウンロード手順**:
```
1. read_page(filter: "interactive") でドロップダウンのrefとPDFリンクのrefを取得
2. form_input(ref=ドロップダウン, value="YYYYMM") で月を選択
3. wait(2秒)
4. read_page で更新後のPDFリンクrefを再取得（refが変わる可能性あり）
5. computer(left_click, ref=PDFリンク) でダウンロード
6. wait(2秒)
7. 次の月へ → 手順2に戻る
```

**注意点**:
- ドロップダウンに存在しない月は未確定（「計算中」）のため、スキップして報告
- PDFファイル名パターン: `SN-YYYYMM-NNNNN.pdf`
- ダウンロード後 `ls -lt ~/Downloads/SN-*.pdf | head` で確認

### 汎用ワークフロー作成ガイド

新しいサイト用ワークフローを作成する際は以下を記録する：
1. **ナビゲーションパス**: URL → クリック対象 → 遷移先の順序
2. **セレクタ情報**: read_pageで取得したref構造、特にフォーム要素
3. **認証ステップ**: どこでユーザー介入が必要か
4. **ループ構造**: 繰り返し操作（月別DL等）のパターン
5. **エラーハンドリング**: タイムアウト、要素未検出時の対応

## トラブルシューティング

### ウェブページが見えない場合
- ページを更新し、拡張機能が有効か確認
- 現在のサイトに権限を付与しているか確認
- 重いJavaScriptのサイトは完全にロードするまで待つ

### アクションが正しく動作しない場合
- 最新版のChromeを使用しているか確認
- ウェブページ操作に干渉する他の拡張機能を無効化
- ページを更新してタスクを再開

### インストール・サインインできない場合
- 有効なMaxプランサブスクリプションがあるか確認
- claude.aiのブラウザキャッシュとCookieをクリア
- Claudeアカウントからサインアウトして再サインイン

### パフォーマンス問題
- 不要なタブを閉じてブラウザリソースを解放
- 複雑なタスクを小さなステップに分割

## 使用制限
使用制限は異なるインターフェース間で共有されるため、Claude for Chromeの使用はClaude.aiやClaude Codeと同じMaxプラン制限にカウントされます。ブラウザ操作は通常のチャットより計算負荷が高いため、より多くの制限を消費します。

## Claude Code との連携
Claude for ChromeはClaude Codeと連携可能。ターミナルで開発し、ブラウザでテスト—両ツールがシームレスに連携します。
