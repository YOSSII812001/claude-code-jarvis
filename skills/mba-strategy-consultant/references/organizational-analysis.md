# 組織分析フレームワーク

## 3.1 McKinsey 7S Model

### 概要
組織の7つの要素の整合性を分析するフレームワーク

```
                    Structure
                       │
                       │
    Strategy ─────── Shared ─────── Systems
                     Values
                       │
                       │
        Style ─────────┼───────── Staff
                       │
                     Skills
```

### 7つの要素

#### ハードS（変更しやすい）

| 要素 | 説明 | 分析の問い |
|------|------|------------|
| **Strategy** | 競争優位を築くための計画 | 戦略は明確か？実行されているか？ |
| **Structure** | 組織図、報告関係、役割分担 | 組織構造は戦略に適合しているか？ |
| **Systems** | 業務プロセス、IT、評価制度 | システムは効率的か？戦略を支援しているか？ |

#### ソフトS（変更が難しい）

| 要素 | 説明 | 分析の問い |
|------|------|------------|
| **Shared Values** | 組織の核となる価値観 | 価値観は浸透しているか？行動に反映されているか？ |
| **Style** | リーダーシップスタイル | リーダーの行動は価値観と一致しているか？ |
| **Staff** | 人材、採用、育成 | 必要な人材がいるか？適切に育成されているか？ |
| **Skills** | 組織の能力、コンピテンシー | 戦略実行に必要なスキルがあるか？ |

### 7S分析テンプレート

| 要素 | 現状 | あるべき姿 | ギャップ | 施策 |
|------|------|------------|----------|------|
| Strategy | | | | |
| Structure | | | | |
| Systems | | | | |
| Shared Values | | | | |
| Style | | | | |
| Staff | | | | |
| Skills | | | | |

### 整合性チェック

**各Sの組み合わせで整合性を確認:**
- Strategy × Structure: 戦略を実行できる組織構造か？
- Strategy × Skills: 戦略に必要なスキルがあるか？
- Structure × Systems: 組織構造をシステムが支援しているか？
- など、すべての組み合わせをチェック

## 3.2 Value Chain Analysis（バリューチェーン分析）

### 概要
マイケル・ポーターが開発した、企業活動を価値創造の連鎖として分析するフレームワーク

```
┌────────────────────────────────────────────────────────────┐
│                    支援活動（Support Activities）          │
├────────────────────────────────────────────────────────────┤
│ 全般管理（インフラ）: 経営企画、財務、法務、品質管理      │
├────────────────────────────────────────────────────────────┤
│ 人的資源管理: 採用、教育、評価、報酬                      │
├────────────────────────────────────────────────────────────┤
│ 技術開発: R&D、製品設計、プロセス改善                     │
├────────────────────────────────────────────────────────────┤
│ 調達: 原材料、設備、サービスの購買                        │
├────────────────────────────────────────────────────────────┤
│                    主活動（Primary Activities）            │
│ ┌────────┬────────┬────────┬────────┬────────┐          │
│ │購買物流│ 製造   │出荷物流│販売・  │サービス│ → 利益 │
│ │        │ オペレ │        │マーケ  │        │   Margin│
│ │Inbound │ーション│Outbound│ティング│        │          │
│ │Logistics│       │Logistics│&Sales │        │          │
│ └────────┴────────┴────────┴────────┴────────┘          │
└────────────────────────────────────────────────────────────┘
```

### 主活動の分析

| 活動 | 内容 | コスト要因 | 差別化要因 |
|------|------|------------|------------|
| 購買物流 | 原材料受入、保管、在庫管理 | | |
| 製造 | 組立、検査、包装、設備保全 | | |
| 出荷物流 | 完成品保管、配送、受注処理 | | |
| マーケティング・販売 | 広告、プロモーション、営業、価格設定 | | |
| サービス | 設置、修理、顧客サポート、トレーニング | | |

### 活用方法

1. **コスト分析** - どの活動にコストがかかっているか
2. **差別化分析** - どの活動で差別化できるか
3. **アウトソーシング判断** - どの活動を外部化すべきか
4. **投資判断** - どの活動に投資すべきか
