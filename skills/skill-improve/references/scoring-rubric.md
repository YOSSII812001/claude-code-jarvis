# スコアリング基準 詳細リファレンス

> このファイルは `/skill-improve` の audit / 個別改善モードで参照される。
> メイン SKILL.md からは `Read references/scoring-rubric.md` で読み込む。

---

## 1. 機械的チェック 10項目

### #1 YAML frontmatter の存在

- **検出**: ファイル先頭が `---` で始まり、2つ目の `---` で閉じられている
- **合格**: frontmatter ブロックが存在する
- **不合格**: frontmatter がない、または不完全

### #2 name フィールド

- **検出**: frontmatter 内に `name:` 行が存在する
- **合格**: `name: <スキル名>` が記述されている
- **不合格**: name フィールドがない

### #3 description にトリガーワード

- **検出**: SKILL.md 内に `トリガー:` または `トリガー：` を含む行がある
- **合格**: トリガーワードのリストが記述されている
- **不合格**: トリガーワードの定義がない
- **備考**: description 内でもスキル本文内でも可

### #4 行数が適正範囲（100-800行）

- **検出**: ファイルの総行数
- **合格**: 100行以上 800行以下
- **注意**: 800行超は冗長の可能性、100行未満は情報不足の可能性
- **備考**: references/ に分離している場合は SKILL.md 単体で判定

### #5 コードブロックの存在

- **検出**: ` ``` ` の出現回数を2で割った数（開始/終了ペア）
- **合格**: 1個以上のコードブロックが存在
- **理想**: 2-4個（実用的な例示）
- **注意**: 0個はリファレンスとして不十分、5個以上は過剰の可能性

### #6 テーブルの存在

- **検出**: `|` を含む行が3行以上連続する箇所の有無
- **合格**: 1つ以上のマークダウンテーブルが存在
- **備考**: 構造化された情報提示に有効

### #7 関連スキルセクション

- **検出**: `## 関連スキル` または `関連スキル` を含む見出し行
- **合格**: 他のスキルへの参照が記述されている
- **備考**: スキル間の連携を明示するために重要

### #8 改訂履歴セクション

- **検出**: `改訂履歴` または `更新履歴` を含む見出し行
- **合格**: 日付と変更内容のリストが存在
- **備考**: スキルの進化を追跡するために重要

### #9 トラブルシューティング

- **検出**: `トラブル` または `エラー` を含む見出し行
- **合格**: よくある問題と対処法が記述されている
- **備考**: マインドセット型スキルでは不要な場合あり（用途適合性で判断）

### #10 チェックリスト

- **検出**: `- [ ]` または `- [x]` を含む行の有無
- **合格**: 手順や確認事項がチェックリスト形式で提供されている
- **備考**: ワークフロー型スキルで特に重要

## 2. 3段階評価の算出

| ランク | 条件 | 意味 |
|--------|------|------|
| **Good** | 10項目中 8個以上クリア | 品質十分。軽微な調整のみ |
| **Needs Work** | 10項目中 5-7個クリア | 特定セクションの強化を推奨 |
| **Critical** | 10項目中 4個以下クリア | 大幅な改善が必要 |

## 3. スキルカテゴリと用途適合性

### カテゴリ定義

| カテゴリ | 説明 | 例 |
|----------|------|-----|
| **リファレンス型** | API・CLI・ライブラリの使い方を参照するスキル | context7, supabase-cli, stripe-cli, vercel-cli |
| **ワークフロー型** | 手順に従って作業を進めるスキル | agent-teams, git-worktree, design-review-checklist |
| **マインドセット型** | 思考法・判断基準を提供するスキル | stoic-daily-practice, wellbeing-mindset, mba-strategy-consultant |
| **プロジェクト型** | 特定プロジェクトの開発ガイド | usacon, usacon-account-mgmt |

### カテゴリ別の重要度調整

| チェック項目 | リファレンス型 | ワークフロー型 | マインドセット型 | プロジェクト型 |
|-------------|--------------|--------------|----------------|--------------|
| YAML frontmatter | 必須 | 必須 | 必須 | 必須 |
| name フィールド | 必須 | 必須 | 必須 | 必須 |
| トリガーワード | 重要 | 重要 | 重要 | 重要 |
| 適正行数 | 重要 | 重要 | やや重要 | 重要 |
| コードブロック | **非常に重要** | 重要 | 不要 | **非常に重要** |
| テーブル | **非常に重要** | 重要 | あれば良い | 重要 |
| 関連スキル | 重要 | 重要 | あれば良い | 重要 |
| 改訂履歴 | 重要 | 重要 | 重要 | 重要 |
| トラブルシューティング | **非常に重要** | 重要 | 不要 | **非常に重要** |
| チェックリスト | あれば良い | **非常に重要** | あれば良い | 重要 |

## 4. 用途適合性分析プロンプトテンプレート

個別改善モード（`/skill-improve <skill-name>`）の Step 4 で使用するサブエージェントプロンプト:

```
あなたはスキル品質アナリストです。以下のスキルファイルを分析し、改善提案を行ってください。

## 対象スキル
{SKILL.md の全文}

## 過去のフィードバック（tasks/lessons.md より）
{当該スキルに関するフィードバック抜粋。なければ「なし」}

## 機械的チェック結果
{10項目のチェック結果}

## 分析タスク

1. **カテゴリ判定**: このスキルは以下のどれに該当しますか？
   - リファレンス型（API・CLI・ライブラリの使い方参照）
   - ワークフロー型（手順に従って作業を進める）
   - マインドセット型（思考法・判断基準を提供）
   - プロジェクト型（特定プロジェクトの開発ガイド）

2. **用途適合性評価**: カテゴリに応じて、以下を評価してください:
   - このスキルの目的は明確か？
   - 目的に対して必要な情報が十分に記述されているか？
   - 冗長な情報はないか？
   - 実際に使う場面で、このスキルだけで作業を完了できるか？

3. **具体的な改善提案**: 以下の形式で列挙してください:
   - 【構造的変更】: スキルの構造や大きなセクションの追加・削除
   - 【軽微な整備】: 表現の修正、セクション順序の調整、欠落情報の補完

4. **優先度**: 最も効果的な改善を3つまで、優先順に提示してください。

出力はマークダウン形式で、簡潔に。
```
