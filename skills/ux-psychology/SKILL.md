# UX Psychology - 心理学に基づくUI/UXデザイン

UI/UX実装時に心理学の原則を適用し、ユーザー体験を向上させるためのガイドライン。

参照元: https://www.shokasonjuku.com/ux-psychology

---

## How to Use This Skill

UIデザインやUX改善のリクエストがあった場合、以下のワークフローに従う：

### Step 1: 要件分析

ユーザーの要求から以下を特定：
- **UI要素**: フォーム、CTA、ナビゲーション、カード、モーダルなど
- **目的**: コンバージョン向上、エンゲージメント向上、信頼性向上など
- **コンテキスト**: ECサイト、SaaS、ダッシュボード、ランディングページなど

### Step 2: 適用可能な原則を選択

下記の「適用ガイド」と「原則一覧」から、要件に合った心理学原則を2-5個選択。

### Step 3: 実装

選択した原則をコードに反映し、コメントで適用した原則を明記。

```tsx
// UX Psychology Applied:
// - 損失回避 (Loss Aversion): 「今だけ」表示で緊急性を演出
// - 社会的証明 (Social Proof): レビュー数を表示
```

## UX心理学 原則一覧（43原則）

### 視覚・認知系

| 原則 | 英語名 | 説明 | 適用例 |
|------|--------|------|--------|
| 美的ユーザビリティ効果 | Aesthetic-Usability Effect | 美しいデザインは使いやすく感じられる | 洗練されたUIコンポーネント |
| 認知負荷 | Cognitive Load | 情報処理に必要な精神的エネルギー | 段階的フォーム、チャンク化 |
| バナー・ブラインドネス | Banner Blindness | 広告らしい要素は無視される | 重要CTAは広告風にしない |
| 選択的注意 | Selective Attention | 目的に沿った情報に集中 | 重要要素を視覚的に強調 |
| 視覚的階層 | Visual Hierarchy | 視覚的優先順位で情報整理 | サイズ、色、位置で階層化 |
| ビジュアル・アンカー | Visual Anchor | 視覚的強調で注意を引く | ハイライト、コントラスト |
| スキューモーフィズム | Skeuomorphism | 現実世界を模倣 | トグル、カードデザイン |

### 意思決定・行動系

| 原則 | 英語名 | 説明 | 適用例 |
|------|--------|------|--------|
| アンカー効果 | Anchor Effect | 最初の情報が基準になる | 元値を先に表示 |
| おとり効果 | Decoy Effect | おとりで他を魅力的に | 料金プランの設計 |
| デフォルト効果 | Default Bias | デフォルトにとどまる傾向 | 推奨設定をデフォルトに |
| フレーミング効果 | Framing | 提示方法が判断に影響 | 「90%成功」vs「10%失敗」 |
| 損失回避 | Loss Aversion | 損失を避けたい傾向 | 「今だけ」「残りわずか」 |
| 決断疲れ | Decision Fatigue | 決断の繰り返しで質低下 | 選択肢を3-5個に絞る |
| 確証バイアス | Confirmation Bias | 既存信念に合う情報を優先 | ユーザー期待に沿うUI |

### エンゲージメント系

| 原則 | 英語名 | 説明 | 適用例 |
|------|--------|------|--------|
| 好奇心ギャップ | Curiosity Gap | 情報欠如が行動を促す | ティーザー、「続きを読む」 |
| ゲーミフィケーション | Gamification | ゲーム要素でモチベ向上 | バッジ、ポイント、レベル |
| 目標勾配効果 | Goal Gradient Effect | ゴール近づくと加速 | プログレスバー |
| 変動型報酬 | Variable Reward | 予測不可な報酬が継続性を高める | ランダム通知、サプライズ |
| ツァイガルニク効果 | Zeigarnik Effect | 未完了が記憶に残る | 進行中表示、チェックリスト |
| 誘惑の結びつけ | Temptation Bundling | 好きな活動と苦手タスク結合 | 報酬付きタスク完了 |

### 信頼・社会系

| 原則 | 英語名 | 説明 | 適用例 |
|------|--------|------|--------|
| 社会的証明 | Social Proof | 他者の行動が信頼性を高める | レビュー、「○人が購入」 |
| ハロー効果 | Halo Effect | 一部印象が全体評価に影響 | 有名ブランドロゴ、信頼バッジ |
| 親近性バイアス | Familiarity Bias | 馴染みのあるものを好む | 一般的UIパターン使用 |
| 段階的要請 | Foot in the Door | 小さな要求→大きな要求 | 段階的オンボーディング |
| 希少性効果 | Scarcity | 限定性が価値を高める | 「在庫残り3個」「期間限定」 |

### 体験評価系

| 原則 | 英語名 | 説明 | 適用例 |
|------|--------|------|--------|
| ピーク・エンドの法則 | Peak-End Rule | 最高と最後で評価 | 印象的な完了画面 |
| 授かり効果 | Endowment Effect | 所有物を過大評価 | 試用期間、カスタマイズ |
| 労働の錯覚 | Labor Illusion | 労力が見えると価値を感じる | ローディング中のプロセス表示 |
| ユーザー歓喜効果 | User Delight | 喜びや驚きの瞬間 | アニメーション、イースターエッグ |
| サンクコスト効果 | Sunk Cost Effect | 過去投資を正当化したい | 進捗可視化、達成バッジ |

### パフォーマンス系

| 原則 | 英語名 | 説明 | 適用例 |
|------|--------|------|--------|
| ドハティの閾値 | Doherty Threshold | 0.4秒以上で興味喪失 | 即時フィードバック、スケルトンUI |
| 段階的開示 | Progressive Disclosure | 情報を段階的に表示 | アコーディオン、詳細展開 |
| 意図的な壁 | Intentional Friction | 意図的に行動を妨げる | 削除確認、二段階認証 |
| プライミング効果 | Priming | 事前刺激が後の行動に影響 | 入力例、プレースホルダー |
| 系列位置効果 | Serial Position Effect | 最初と最後が記憶されやすい | 重要項目を先頭・末尾に |

## コード実装時のコメント記法

```tsx
/**
 * UX Psychology Applied:
 * - [原則名] ([英語名]): [適用理由]
 * - [原則名] ([英語名]): [適用理由]
 */
```

### 例

```tsx
/**
 * UX Psychology Applied:
 * - 損失回避 (Loss Aversion): 「今だけ」表示で緊急性を演出
 * - 希少性効果 (Scarcity): 残り在庫数を表示して購買意欲を刺激
 * - 社会的証明 (Social Proof): レビュー件数を表示して信頼性向上
 */
export function ProductCard({ product }: { product: Product }) {
  return (
    <div className="card">
      {/* 希少性効果: 在庫数表示 */}
      {product.stock < 5 && (
        <Badge variant="warning">残り{product.stock}個</Badge>
      )}

      {/* 社会的証明: レビュー数 */}
      <div className="reviews">
        <Stars rating={product.rating} />
        <span>({product.reviewCount}件のレビュー)</span>
      </div>

      {/* 損失回避: 期間限定表示 */}
      {product.isLimitedOffer && (
        <Badge variant="destructive">今だけ特別価格</Badge>
      )}
    </div>
  )
}
```

## トラブルシューティング

### Q: どの心理学原則を適用すべきか判断できない
**A:** まず「適用ガイド（UI要素別）」セクションを参照してください。対象のUI要素（フォーム、CTA、料金プランなど）に対応する推奨原則が一覧化されています。迷う場合は、認知負荷の軽減と社会的証明の2つから始めるのが汎用的に効果的です。

### Q: 心理学原則の適用がダークパターンにならないか心配
**A:** ユーザーの利益と一致する方向で原則を適用しているか確認してください。例えば「損失回避」は本当に有益な情報の見逃しを防ぐために使うのは良いですが、偽の緊急性を煽るのはダークパターンです。「ユーザーがこの仕組みを知っても納得できるか？」を判断基準にしてください。

### Q: 複数の原則が矛盾する場合はどうするか
**A:** 例えば「認知負荷の軽減（情報を減らす）」と「社会的証明（レビュー数を表示する）」が競合する場合は、段階的開示を活用して初期表示は最小限にし、詳細は展開可能にすることで両立できます。優先すべきはユーザーの主タスク達成です。
